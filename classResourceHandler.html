template<typename Key , typename Resource >
int ResourceHandler< Key, Resource >::dereference ( Key const &  key)
inline

Marks that the work with a reference to a previously requested resource is done.

Remarks
Programmer should not expect the dereferenced resource to work properly. Call again get(Key const& key) to recover access. Be aware that before you try to get a reference back the resource will already be deleted if no references where left and flush() was called.
Parameters
keyKey previously used to add a resource.
Returns
Number of references left to the resource after dereferencing. -1 if there's no resource with the given key.
